C++ Hash Table

Once I learned about hashs tables, I thought they were the coolest thing. You could access data immediately! Why would you not use hash tables everywhere? But there are drawbacks as I learned, like collisions. Anyway I wanted to practice making hash tables, so I made a word unscrambler with a hash table.

A word unscrambler is the thing you use to cheat in Words with Friends and scrabble. It takes in a set of letters (like "abc") and outputs a list of possible words to create from them (like CAB, DAB, and BAD). A hash table is great for this application because we need to compare permutations of the letters against all words to see if they match. All these comparisons need to be fast, and a hash table is fast for lookups. üòÅ

So I begun by downloading the first available dictionary I found on the internet. I got one with 236k words, and now needed to engineer the hash table that will be my dictionary. I decided to go with chaining collision resolution. If you want to know the math behind it then look at the presentation linked below, but the end table had an average lookup of 8 words. This means when I want to compare one permutation of the input string (like BAD), then I only need to compare it against 8 other words on average. In the computer science world, lightning quick. ‚ö°
